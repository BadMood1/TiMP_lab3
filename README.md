# Лабораторная работа 2: Реализация программы, управляемой данными

## Описание проекта

Проект демонстрирует реализацию программы, управляемой данными, с использованием C# и Avalonia UI. Программа включает систему авторизации и динамическое построение меню на основе внешних текстовых файлов.

## Структура проекта

- **AuthorizationLibrary**: Библиотека для авторизации пользователей и управления правами доступа.
- **DataMenuLibrary**: Библиотека для чтения и парсинга структуры меню из файла.
- **MenuExplicitApp**: Приложение с явным управлением (динамическая загрузка библиотек без прямых ссылок).
- **MenuImplicitApp**: Приложение с неявным управлением (прямые ссылки на библиотеки).
- **menu.txt**: Файл с данными структуры меню.
- **USERS.txt**: Файл с данными пользователей и их прав доступа.

## Требования к реализации

Программа, управляемая данными, должна соответствовать следующим требованиям:

1. **Внешнее хранение данных**: Структура меню и права пользователей хранятся в текстовых файлах (`menu.txt`, `USERS.txt`).
2. **Динамическое построение**: Меню строится на основе данных из файлов без жесткого кодирования в программе.
3. **Авторизация**: Система входа с проверкой логина/пароля и контролем доступа к пунктам меню.
4. **Уровни доступа**: 
   - 0: Виден и доступен
   - 1: Виден, но недоступен
   - 2: Скрыт
5. **Модульность**: Разделение на библиотеки для авторизации и данных меню.
6. **Два режима реализации**:
   - **Explicit**: Динамическая загрузка библиотек через рефлексию.
   - **Implicit**: Прямое использование классов библиотек.
7. **Обработчики действий**: Каждый пункт меню может иметь связанный метод для выполнения действия.

## Запуск проекта

1. Убедитесь, что установлены .NET 9.0 и Avalonia.
2. Соберите решение: `dotnet build DataDrivenMenu.sln`
3. Запустите нужное приложение:
   - Для explicit: `dotnet run --project MenuExplicitApp`
   - Для implicit: `dotnet run --project MenuImplicitApp`

## Формат файлов данных

### menu.txt
Формат: `уровень название_пункта [статус] [имя_метода]`

Пример:
```
0 Разное 0 Others
1 Отделы 0 Departs
```

### USERS.txt
Формат:
```
#username password
пункт_меню уровень_доступа
```

Пример:
```
#Boss 1234
Справочники 2
Отделы 2
```
